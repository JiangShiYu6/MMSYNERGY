# 基于相似性图去噪改进宏观编码器——方案汇报

---

## 一、研究动机

### 1.1 问题发现

我们对 MMFSynergy 的宏观异构网络做了**扰动鲁棒性测试**——随机删除一定比例的边，然后重新训练 MacroEncoder 做链接预测，观察性能变化。

| 边去除比例 | AUC | 相比完整图的变化 |
|-----------|------|-----------------|
| 0%（完整图）| 0.8784 | — |
| 5% | 0.8843 | +0.67% |
| 10% | 0.8765 | -0.22% |
| 20% | **0.8860** | **+0.86%** |
| 30% | 0.8722 | -0.71% |

**正常预期**：删边等于删信息，性能应该单调下降。

**实际表现**：删掉 20% 的边后，AUC 反而达到最高。

### 1.2 问题诊断

这说明原始图中存在**噪声边**——不反映真实生物学关系的虚假连接。这些边在 GNN 的消息传递过程中会让不相关的节点互相影响，**污染**节点表示。随机删边时恰好删掉一些噪声边，模型反而学得更好。

## 二、文献调研

我们阅读了四篇相关论文，逐一分析与当前问题的匹配程度。

### 2.1 论文概要

**论文1：HGNN-WI**（Expert Systems With Applications）

- 全称：Heterogeneous Graph Neural Networks with Weak Information
- 核心方法：metapath 子图补全 + kNN 增强图 + 多视角对比学习
- 解决的问题：异构图中信息**缺失**（结构不完整、特征缺失、标签稀疏）

**论文2：NoiseHGNN**（AAAI 2025）

- 全称：Synthesized Similarity Graph-Based Neural Network For Noised Heterogeneous Graph Representation Learning
- 核心方法：相似性图合成 + 相似性感知注意力 + Metapath-Target 对比学习
- 解决的问题：异构图中的**噪声边**（错误链接、结构错误）

**论文3：HGNN-AR2**（Neural Networks 2025）

- 全称：Heterogeneous Graph Neural Network with Adaptive Relation Reconstruction
- 核心方法：图自编码器重建边概率 + 插值融合 + Gumbel-Softmax Bernoulli 采样
- 解决的问题：metapath 连接中的类间错误连接和类内连接缺失

**论文4：SAGNN**（Information Fusion）

- 全称：Multi-level Feature Subgraph Aggregation Graph Neural Network
- 核心方法：同构/异构子图分离 + 多层级特征聚合 + 加权融合
- 解决的问题：多层级特征提取不充分、忽略同构/异构邻居交互差异

### 2.2 匹配度分析

| 论文 | 解决的问题 | 与我们的匹配度 | 理由 |
|------|-----------|--------------|------|
| 论文1 HGNN-WI | 信息**缺失** | 中 | 方向偏了，我们是信息**有噪声**而非缺失 |
| **论文2 NoiseHGNN** | **噪声边** | **高** | **问题完全对应，直接解决异构图噪声边问题** |
| 论文3 HGNN-AR2 | 连接不准确 | 中高 | 可过滤噪声，但设计基于 metapath 场景，需要较多适配 |
| 论文4 SAGNN | 特征提取不充分 | 低 | 不处理噪声问题 |

### 2.3 选择结论

**论文2（NoiseHGNN，AAAI 2025）**最适合，原因：

1. 它专门解决异构图噪声边问题，与我们的数据质量问题完全对应
2. 它在 30% 噪声率的 5 个数据集上验证了有效性，我们的数据噪声率估计在 15-25%（从扰动实验推断），在它的有效范围内
3. NoiseHGNN 在 DBLP 和 IMDB 上比同期 SOTA 方法提升了 3-6%

## 三、提出的改进方案

### 3.1 当前方法的问题

现有的 `MacroEncoder` 结构如下：

```
节点特征 → 线性投影 → HANLayer × 2 → 节点嵌入 → 链接预测
               ↑
         GATConv 对所有边一视同仁
         噪声边和真实边同等参与消息传递
```

HANLayer 的工作方式是：通过 GATConv 在每种边类型上做图注意力卷积，然后用语义注意力聚合不同边类型的结果。问题在于，GATConv 只根据节点特征学注意力权重，**无法判断一条边本身是否是噪声**。噪声边和真实边以同等地位参与消息传递，导致节点表示被污染。

### 3.2 改进思路：引入相似性图作为去噪信号

核心想法来自 NoiseHGNN：

> **如果两个节点特征相似，它们之间的连接更可能是真实的；如果特征差异很大却有边相连，这条边很可能是噪声。**

我们利用这个先验知识，构建一张独立于原始拓扑的"**相似性图**"，作为去噪的参考信号。

### 3.3 具体架构

改进后的 `DenoisedMacroEncoder` 由三个部分组成。

#### Part 1：相似性图构建器（SimilarityGraphBuilder）

```
对每种节点类型（drug / protein / sideeffect）：

  1. 节点特征 X → 线性投影 → Z
  2. Z 做 L2 归一化
  3. 计算余弦相似度矩阵 S = Z × Z^T
  4. kNN 稀疏化：每个节点只保留 top-k（默认15）最相似的邻居
  5. 去除自环
  → 得到相似性邻接矩阵 A_sim
```

这张相似性图有一个关键性质：**它完全基于节点特征构建，不依赖原始图的边信息**。所以即使原始图有噪声边，相似性图也不会被污染。相似性图提供了一个独立的、纯粹基于生化特征的节点关系视角。

#### Part 2：双路径编码

两条并行的编码路径同时处理节点信息：

```
路径 A（原始图路径）：
  投影后的节点特征 → HANLayer × 2（在原始异构图上）→ H_orig
  ✓ 保留了原始图的丰富拓扑信息和异构关系语义
  ✗ 也包含了噪声边的影响

路径 B（相似性图路径）：
  投影后的节点特征 → 归一化邻接矩阵 A_sim × 特征 → MLP → H_sim
  ✓ 只传递相似节点之间的信息，天然过滤了噪声连接
  ✗ 缺少异构关系语义（drug-protein 等不同边类型的区分）
```

两条路径互为补充：原始图路径提供异构语义，相似性图路径提供去噪信号。

#### Part 3：融合与对比学习

**表示融合**：

```
H_fused = α × H_orig + (1 - α) × H_sim
```

其中 α 默认取 0.7。原始图虽然有噪声，但它包含了真实的异构关系语义（drug-protein、protein-protein 等不同类型的边），这是相似性图无法提供的。所以原始图仍然是主要信息来源（权重 0.7），相似性图只是辅助校正（权重 0.3）。

**对比学习正则化损失**：

```
L_contrastive = InfoNCE(H_orig, H_sim)
```

对比学习鼓励同一节点在两条路径中的表示保持一致，不同节点的表示保持差异。这迫使模型找到两个视角的**共识**——真实的连接关系在两个视角中都会得到支持，而噪声连接只出现在原始图中，在对比学习的压力下会被自动降权。

**总损失函数**：

```
L_total = L_link_pred + λ × L_contrastive    （λ 默认 0.1）
```

### 3.4 改进前后架构对比

**改进前（标准 MacroEncoder）**：

```
原始异构图 → 线性投影 → HANLayer × 2 → 节点嵌入 → 链接预测头 → BCELoss
```

**改进后（DenoisedMacroEncoder）**：

```
原始异构图 ──→ 线性投影 → HANLayer × 2 ────→ H_orig ──┐
                                                        ├→ 加权融合 → H_fused → 链接预测头 → BCELoss
节点特征 ───→ 相似性图构建器                              │                                      ↓
              (余弦相似度 + kNN)                          │                              L_link_pred
              → 归一化邻接 × 特征 → MLP ──→ H_sim ──────┘                                      +
                                              │                                          λ × L_contrastive
                                    对比学习损失 ←──── H_orig                                    ↓
                                                                                          L_total
```

## 四、理论分析：为什么这个方案能解决问题

从三个层面分析方案的可行性。

### 4.1 信号层面

噪声边连接的两个节点，在生物学上不存在真实交互关系，因此它们的生化特征（SMILES 分子式、氨基酸序列等编码得到的特征向量）通常不相似。在相似性图中，这些节点对的余弦相似度较低，不会进入 top-k 邻居，因此不会有连接。通过融合相似性图的表示，噪声边对节点嵌入的影响被稀释。

### 4.2 信息论层面

原始图和相似性图提供了**两个独立的视角**看同一组节点：
- 原始图视角：基于已知的生物网络拓扑
- 相似性图视角：基于节点的内在特征相似性

对比学习迫使模型找到两个视角的**共识**。真实的连接关系在两个视角中都会得到支持（真实交互的节点特征通常也相似），而噪声连接只出现在原始图中、在相似性图中没有对应。在对比学习的压力下，模型会自动降低对这些不一致连接的依赖。

### 4.3 实验先例

NoiseHGNN 在以下条件下验证了这个思路的有效性：
- 30% 噪声率的 5 个数据集（DBLP、IMDB、ACM、PubMed、Freebase）
- 在 DBLP 和 IMDB 上比同期 SOTA 方法提升 3-6%
- 消融实验表明相似性图构建器和对比学习模块各自都有独立贡献

我们的数据噪声率估计在 15-25%（从"删 20% 边性能最优"推断），在 NoiseHGNN 的有效范围内。

## 五、实验设计

### 5.1 核心实验：扰动鲁棒性对比

用**相同的扰动率**（0%、5%、10%、20%、30%），分别训练 Baseline（标准 MacroEncoder）和 Denoised（去噪 MacroEncoder），对比 AUC 和 AP。

**判断标准**：
1. 去噪模型在 0% 扰动时 AUC 是否**高于** Baseline → 说明去噪有效
2. 去噪模型的性能曲线是否更接近**单调下降** → 说明数据质量问题被缓解
3. 去噪模型在各个扰动率下的 AUC 是否**整体高于** Baseline → 说明表示质量全面提升

### 5.2 消融实验

为了验证每个组件的独立贡献，设计以下消融配置：

| 配置 | 相似性图路径 | 对比学习 | 说明 |
|------|:-----------:|:-------:|------|
| Baseline | ✗ | ✗ | 标准 MacroEncoder，作为基线 |
| + SimGraph only | ✓ | ✗ | 只加相似性图路径和融合，验证去噪信号的价值 |
| + Contrastive only | ✗ | ✓ | 标准 HAN + 对比学习，验证正则化的价值 |
| Full Model | ✓ | ✓ | 相似性图 + 对比学习，完整方案 |

### 5.3 超参数敏感性分析

关键超参数及其搜索范围：

| 超参数 | 含义 | 默认值 | 搜索范围 |
|--------|------|--------|---------|
| k | kNN 相似性图的邻居数 | 15 | {5, 10, 15, 25, 35} |
| α | 原始图路径的融合权重 | 0.7 | {0.5, 0.6, 0.7, 0.8, 0.9} |
| λ | 对比学习损失的权重 | 0.1 | {0.01, 0.05, 0.1, 0.2, 0.5} |

| 文件 | 改动类型 | 说明 |
|------|---------|------|

| SynergyBert 相关 | **不修改** | train_main_macro.py, train_multitask_synergy_macro.py 等 |

- `SimilarityGraphBuilder`：相似性图构建器
- `DenoisedMacroEncoder`：去噪宏观编码器（含双路径编码、融合、对比学习）
