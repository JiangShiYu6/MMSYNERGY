# 基于自适应关系重建（HGNN-AR2）改进 MMFSynergy 宏观编码器——方案设计

---

## 一、论文3（HGNN-AR2）方法回顾

### 1.1 原始流程

论文3针对 metapath 构建的邻接矩阵中存在的**不准确连接**和**缺失连接**问题，提出了自适应关系重建方法。核心流程如下：

```
Step 1: 对每种 metapath 邻接矩阵 A_i，用 GCN 编码节点
        Z_i = GCN(X, A_i)

Step 2: 内积解码，预测边概率
        Ã_i = Norm(Z_i × Z_i^T)        ← N×N 稠密矩阵

Step 3: 与原始邻接插值融合
        P_i = α·Ã_i + (1-α)·A_i

Step 4: Gumbel-Softmax Bernoulli 采样，稀疏化
        Â_i = GumbelSample(P_i, τ)

Step 5: 在重建后的图上做 GCN 分类
        H_i = GCN(X, Â_i)

Step 6: 多 metapath 加权融合 + 联合损失
        L = L_ce + β·L_rec
```

### 1.2 核心思想

**不直接依赖原始特征的相似度**，而是先通过 GCN 在原始图上传播特征，让节点嵌入 Z 吸收拓扑信息，然后用 Z 的内积来判断"这条边是否应该存在"。这意味着即使原始特征是随机的，经过图传播后 Z 也能携带有意义的结构信息。

## 二、MMFSynergy 宏观编码器现状

### 2.1 图数据概况

| 节点类型 | 数量 | 特征 |
|---------|------|------|
| Drug | 6,295 | 204维实数特征（drug_feat.npy），269个节点含 NaN |
| Protein | 16,065 | **随机 256 维向量**（无真实特征） |
| Sideeffect | 5,269 | **随机 256 维向量**（无真实特征） |

| 边类型 | 边数 | 密度 | 占比 |
|--------|------|------|------|
| DDI（drug-drug） | 1,070,562 | 2.70% | 76.6% |
| PPI（protein-protein） | 217,160 | 0.08% | 15.5% |
| DSI（drug-sideeffect） | 96,654 | 0.29% | 6.9% |
| DTI（drug-protein） | 12,722 | 0.01% | 0.9% |

**关键观察**：
- DDI 占了 76.6% 的边，密度最高（2.7%），是最可能包含噪声的部分
- DTI 只有 12,722 条边，密度极低（0.01%），这些边非常稀疏和珍贵
- Protein 和 Sideeffect 没有真实特征

### 2.2 当前架构

```
节点特征 → 线性投影(per type → 128维) → HANLayer × 2 → 节点嵌入 → 链接预测头 → BCELoss
```

### 2.3 数据质量问题

删除 20% 边后链接预测 AUC 从 0.8784 升至 0.8860。可能原因：
1. 噪声边干扰消息传递
2. 删边起到正则化效果（类似 DropEdge），防止模型过拟合于度信息等结构捷径

## 三、适配分析：论文3能否用于 MMFSynergy

### 3.1 需要解决的适配问题

| 问题 | 论文3 | MMFSynergy | 适配方案 |
|------|-------|------------|---------|
| 图结构单元 | Metapath 邻接矩阵 | 边类型邻接矩阵 | 按边类型分别重建 |
| 下游任务 | 节点分类 | 链接预测 | 重建损失 + 链接预测损失 |
| 节点特征 | 全部有意义 | Protein/SE 是随机的 | 依赖 GCN 传播获取拓扑特征 |
| 邻接矩阵形状 | 同类型 N×N | 跨类型 N_src×N_dst | 源/目标分别编码后内积 |
| N×N 计算量 | 小图（3K-5K 节点） | 大图（Protein 16K） | **不生成稠密矩阵，只对已有边评分** |

### 3.2 关键适配：避免 N×N 稠密矩阵

论文3对小图（3K-5K 节点）直接算 N×N 内积矩阵。但 MMFSynergy 的 protein 有 16,065 个节点，Protein×Protein 矩阵就需要约 1GB 内存，不现实。

**解决方案**：我们不需要完整的 N×N 矩阵。我们的目的是给**已有的边**打置信度分数，决定保留还是丢弃。所以只需要：

```
对每条已有边 (u, v)：
  score(u, v) = sigmoid(Z_u · Z_v)    ← 只算已有边，不算全部节点对
```

这样计算量从 O(N²) 降到 O(|E|)，即只需对 140 万条边计算分数，完全可行。

### 3.3 GCN 传播能缓解随机特征问题

Protein 和 Sideeffect 的初始特征是随机的，但 GCN 编码器会在图上做消息传递：

```
Z = GCN(X_random, A)
```

经过传播后，Z 不再是随机的——**拓扑位置相近的节点**会有相似的 Z（因为它们聚合了相同邻居的信息）。所以内积 Z_u·Z_v 实际衡量的是"u 和 v 在图拓扑上是否相近"，这是有意义的信号。

当然，如果有真实的 Protein/Sideeffect 特征会更好，但目前条件下 GCN 传播至少能提供拓扑层面的边置信度。

## 四、改进方案详细设计

### 4.1 整体架构

```
                    ┌──────────────────────────────┐
                    │   Edge Confidence Scorer      │
                    │   (轻量 GCN + 内积解码)        │
                    │                              │
原始异构图 G ──────→│   对每种边类型：                │
                    │   Z = GCN(feat, A_etype)     │
                    │   score(u,v) = σ(Z_u · Z_v)  │
                    └──────────┬───────────────────┘
                               │ 每条边的置信度分数
                               ▼
                    ┌──────────────────────────────┐
                    │   Edge Filtering              │
                    │   融合原始边权 + 预测置信度      │
                    │   p = α·score + (1-α)·1.0    │
                    │   Gumbel-Softmax 采样         │
                    └──────────┬───────────────────┘
                               │ 过滤后的图 G'
                               ▼
                    ┌──────────────────────────────┐
                    │   HAN Encoder                 │
                    │   (与现有 MacroEncoder 相同)    │
                    │   HANLayer × 2                │
                    └──────────┬───────────────────┘
                               │ 节点嵌入 H
                               ▼
                    ┌──────────────────────────────┐
                    │   Link Prediction Heads       │
                    │   (与现有相同)                  │
                    └──────────┬───────────────────┘
                               │
                               ▼
                    L_total = L_link_pred + β·L_rec
```

### 4.2 模块1：Edge Confidence Scorer

为每种边类型训练一个轻量级的边置信度评分器。

**同类型边**（drug-drug, protein-protein）：

```python
# 源和目标是同一类型，共享编码器
Z_drug = GCN_dd(drug_feat, A_dd)        # (N_drug, d)
# 对每条已有边 (u, v) 计算分数
score_dd = sigmoid(Z_drug[u] * Z_drug[v])  # dot product per edge
```

**跨类型边**（drug-protein, sideeffect-drug）：

```python
# 源和目标是不同类型，分别编码
Z_drug_dp = GCN_dp_src(drug_feat, A_dp)      # (N_drug, d)
Z_prot_dp = GCN_dp_dst(protein_feat, A_dp.T)  # (N_protein, d)
# 对每条已有边 (u, v) 计算分数
score_dp = sigmoid(Z_drug_dp[u] * Z_prot_dp[v])
```

**GCN 编码器的设计**：

```python
class EdgeTypeGCN(nn.Module):
    """每种边类型一个轻量 GCN，1-2层即可"""
    def __init__(self, in_dim, hidden_dim):
        self.conv1 = GraphConv(in_dim, hidden_dim)
        self.conv2 = GraphConv(hidden_dim, hidden_dim)

    def forward(self, feat, adj):
        z = F.relu(self.conv1(adj, feat))
        z = self.conv2(adj, z)
        return z
```

### 4.3 模块2：Edge Filtering

将边置信度分数与原始边权融合，然后采样。

**Step 1：插值融合**

```python
# score: 模型预测的边置信度 (0~1)
# 原始边默认权重为 1.0
p = alpha * score + (1 - alpha) * 1.0
```

其中 α 控制重建信号的强度。α 过大会过度改变图结构，α 过小则重建无效果。

**Step 2：Gumbel-Softmax 采样**

```python
def gumbel_softmax_sample(p, tau=1.0):
    """可微分的 Bernoulli 采样"""
    G = -torch.log(-torch.log(torch.rand_like(p)))          # Gumbel noise
    y_soft = torch.sigmoid((torch.log(p + 1e-8) + G) / tau) # soft sample
    y_hard = (y_soft > 0.5).float()                          # hard decision
    return y_hard - y_soft.detach() + y_soft                 # straight-through
```

采样结果是每条边的 0/1 决策（前向传播时是离散的，反向传播时梯度通过 soft 值传递）。

**另一种选择：Soft Weighting**

如果不想做硬采样（丢弃边），也可以直接用 p 作为边权重传入 GATConv：

```python
# 在 HAN 的 GATConv 中，用 p 加权边
# 高置信度的边权重大，低置信度的边权重小
```

这种方式更平滑，不会完全丢弃边，适合边数较少的类型（如 DTI 只有 12,722 条）。

### 4.4 模块3：门控融合（替代固定 α）

不使用固定的 α，而是用门控网络让模型自适应决定融合比例。

```python
class GatedFusion(nn.Module):
    def __init__(self, hidden_dim):
        self.gate = nn.Sequential(
            nn.Linear(hidden_dim * 2, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1),
            nn.Sigmoid()
        )

    def forward(self, H_original, H_reconstructed):
        gate_input = torch.cat([H_original, H_reconstructed], dim=-1)
        alpha = self.gate(gate_input)  # (N, 1)，每个节点一个权重
        return alpha * H_original + (1 - alpha) * H_reconstructed
```

这样模型可以自适应判断：对于每个节点，应该更相信原始图的表示还是重建图的表示。

### 4.5 损失函数

**联合损失**：

```
L_total = L_link_pred + β · L_rec
```

**L_link_pred**（主任务，与现有相同）：

```python
# 链接预测的 BCE 损失
# 正样本：图中存在的边
# 负样本：随机采样的不存在的边
L_link_pred = BCELoss(pred_pos, 1) + BCELoss(pred_neg, 0)
```

**L_rec**（重建损失，新增）：

```python
# 鼓励边置信度评分器的输出与原始图一致
# 对已有边：score 应接近 1
# 对随机负样本边：score 应接近 0
L_rec = BCELoss(score_pos_edges, 1) + BCELoss(score_neg_edges, 0)
```

重建损失的意义：防止边过滤模块过于激进地删边，保持与原始图结构的一致性。β 控制重建约束的强度。

## 五、与论文2方案的对比

| 维度 | 论文2方案（相似性图） | 论文3方案（边重建） |
|------|-------------------|-------------------|
| 边置信度来源 | 原始特征余弦相似度 | GCN 传播后的拓扑嵌入内积 |
| 随机特征可用？ | 不可用（随机特征无相似度可言） | **可用**（GCN 传播能学到拓扑信息） |
| 跨类型边处理 | 困难（跨类型相似度无意义） | **可行**（分别编码后内积） |
| 计算量 | N×N 相似度矩阵 | **只算已有边**，O(E) |
| 融合方式 | 固定权重 α | **门控网络自适应** |
| 端到端训练 | 是 | 是 |

## 六、前置实验：区分噪声 vs 正则化

在实施论文3方案之前，先做一个简单实验确认问题的性质。

### 6.1 实验设计

| 实验 | 训练时的图 | 评估时的图 | 目的 |
|------|-----------|-----------|------|
| A. Baseline | 完整图 | 完整图 | 基线 |
| B. DropEdge | 每 epoch 随机丢 20% 边 | 完整图 | 检验正则化效应 |
| C. 固定删边 | 固定删 20% 边 | 剩余图 | 检验噪声效应 |

### 6.2 结果解读

- **B >> A，C ≈ A**：正则化为主 → 直接加 DropEdge 即可
- **C >> A，B ≈ A**：噪声为主 → 需要论文3的边重建
- **B > A 且 C > A**：两者兼有 → 论文3方案 + DropEdge

### 6.3 为什么要先做这个

- 只需改训练代码几行，不改模型架构
- 10 分钟内可以得到结果
- 如果是纯正则化效应，加 DropEdge 就够了，不需要复杂的边重建

### 7.1 阶段一：前置实验（1天）

如果确认需要边重建：

### 7.3 阶段三：实验验证（2-3天）

1. **扰动鲁棒性对比**：相同扰动率下对比 Baseline vs AR2MacroEncoder
2. **消融实验**：
   - w/o Edge Scorer（去掉边评分，只用 DropEdge）

3. **超参数敏感性**：
   - β（重建损失权重）：{0.01, 0.05, 0.1, 0.2, 0.5}
   - τ（Gumbel 温度）：{0.1, 0.5, 1.0, 2.0}

## 八、风险与备选

### 8.1 风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| GCN 传播后随机特征仍无意义 | 边评分不准确 | 对 Protein/SE 只做拓扑层面的评分（1层GCN足够） |
| DTI 边数太少（12,722） | 不宜做硬采样丢弃 | 对稀疏边类型用 soft weighting 而非硬采样 |
| DDI 边数太多（107万） | 计算量大 | 分批计算边评分，或只对训练集边评分 |
| 重建损失与链接预测损失冲突 | 训练不稳定 | 调节 β，或让 L_rec 只在前期生效（warmup） |

### 8.2 备选方案

如果前置实验表明主要是正则化效应：
- 直接在 HAN 训练时加 DropEdge
- 不需要论文3的边重建
- 实现成本极低（几行代码）
